1. NATIVE FUNCTIONS

Native functions are built-in SA-MP functions.

They can be called normally as statements or inside expressions.

The compiler automatically converts friendly SAMPL function names
into the correct Pawn native names.

It also automatically adds "&" references for natives that require
reference parameters in Pawn (such as "GetPos" or "GetHealth").

You do NOT need to manually write "&" for mapped natives.

Mapped native :

GiveMoney(id, amount)  -> GivePlayerMoney(id, amount)

Special note about "GetName":

"GetPlayerName" in Pawn does NOT directly return a string.

Because of that, "GetName()" may ONLY be used inside declarations:

let name : string[24] = GetName(id)

which automatically becomes:

new name[24];
GetPlayerName(id, name, 24);

Using "GetName()" directly inside expressions is NOT supported.

Invalid example:

Send(id, GetName(id))

Correct approach:

let name : string[24] = GetName(id)
Send(id, name)

Special note about "Send":

Send(id, text)

uses the default green color:

0x00FF00FF

Custom colors are also supported:

Send(id, 0xFF0000FF, "Error")

2. CUSTOM FUNCTIONS & UNMAPPED NATIVES

SAMPL is NOT limited to the built-in callback and native tables.

Any function name that does NOT exist inside the callback/native
mapping tables is automatically passed directly into the generated
Pawn file unchanged.

This means:

- Unmapped SA-MP natives such as "CreateVehicle",
  "CreateObject", "TextDrawCreate", etc. can already be used
  immediately without updating the compiler.
- Your own helper functions also work automatically.

Example:

PlayerSpawn(id)
    CreateVehicle(411, 1958.3, 1343.1, 15.3, 0.0, -1, -1, -1)
    return true

Important note:

If an unmapped native uses reference parameters ("&" in Pawn),
you must manually add "&" yourself because the compiler only
automatically inserts references for mapped natives.

3. ADDING CUSTOM NATIVES TO THE COMPILER

Even if a native is not mapped yet, you can already use it through
passthrough mode (see section 10).

However, if you want shorter aliases such as:

GetCash()

instead of:

GetPlayerMoney()

you can edit the "NATIVES" dictionary inside "smplc.py".

Example:

'GetCash': dict(pawn='GetPlayerMoney'),

If the native requires reference parameters ("&" in Pawn), add a
"refs" list containing the argument indexes that need references.

Example:

'GetHealth': dict(
    pawn='GetPlayerHealth',
    refs=[1]
),

To add custom callbacks, edit the "CALLBACKS" dictionary:

'NameInSAMPL': 'OriginalPawnCallback',

No rebuilding or recompilation is required after editing.

"smplc.py" is a normal Python script and changes apply immediately
after saving the file.
